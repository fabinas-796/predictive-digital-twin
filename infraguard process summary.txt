📚 Section 2 — What You Learned in Week 3
1. Flask Web Framework
What Flask is:
Flask is a Python library that turns your Python code into a web server. When you run python app.py, Flask opens a port (5000) on your computer and starts listening for HTTP requests from browsers or other programs.
Routes:
A route is a URL pattern mapped to a Python function. When someone visits /api/overview, Flask calls the api_overview() function and returns whatever it returns. You define routes using the @app.route() decorator.
Two types of routes you built:

Page routes → return render_template('page.html') which serves an HTML file
API routes → return jsonify({...}) which sends JSON data

Why debug=False and use_reloader=False:
These must be False when running background threads. If debug=True, Flask runs your code twice which would start 2 collectors and 2 AI engines. Always False in InfraGuard AI.

2. HTTP Methods

3. Sessions and Authentication
What a session is:
When you log in, Flask creates a session — a small encrypted piece of data stored in a cookie on your browser. Every request you make after that carries this cookie. The server reads it to know who you are without asking you to log in again every time.
What a cookie is:
A cookie is a small text file stored in your browser. In our curl tests, we saved it to cookies.txt with -c and sent it back with -b. A browser does this automatically.
The login_required decorator:
This is a Python decorator — a function that wraps another function. When you put @login_required above an API function, it checks if user is in the session before running the function. If not, it redirects to login. This protects every single API and page automatically.
Role-based access:

Admin → can access everything including /api/admin/users and simulation
Engineer → can access monitoring + simulation but not user management
Viewer → read-only, cannot trigger simulations or manage alerts


4. REST API Design
What REST means:
REST is a standard way of designing APIs. The key rules are:

Use URLs to identify resources (/api/alerts/5 = alert number 5)
Use HTTP methods to define actions (GET = read, POST = create/action)
Return JSON data
Be stateless (each request carries all needed info)

Why APIs matter:
The dashboard JavaScript will call these APIs every few seconds to update the charts and tables without refreshing the page. This is called AJAX — the page stays loaded but data updates dynamically.

5. Background Threading in Flask
The problem:
Flask normally handles one request at a time. But InfraGuard AI needs to collect data every 3 seconds and run AI predictions every 15 seconds — even when nobody is making requests.
The solution — background threads:
We use Python's threading.Thread to run the collector and prediction loop in the background. daemon=True means these threads stop automatically when the main program stops. This is why you see data updating even when you're not doing anything.
Two background threads running at all times:

Collector thread → saves metrics to database every 3 seconds
Prediction loop → runs all 4 AI models every 15 seconds


6. PDF Generation with ReportLab
What ReportLab is:
ReportLab is a Python library for creating PDF files programmatically. Instead of designing a PDF visually, you build it in code by defining elements like Paragraphs, Tables, and Spacers and adding them to a "story" list.
Key concepts:

SimpleDocTemplate — the PDF document container
story — a list of elements that get placed top to bottom
Table — grid of data with custom styling per cell
TableStyle — defines background colours, text colours, fonts, padding per cell range
BytesIO — stores the PDF in memory as bytes before sending it as a download
ParagraphStyle — defines custom text formatting (font size, colour, alignment)

Why dark background:
The _draw_background function draws a dark rectangle across the entire page before each page's content. This gives the professional dark theme matching the dashboard.

7. CSV Generation
What CSV is:
CSV stands for Comma Separated Values. It's the simplest data format — every row is a line, every column is separated by a comma. Opens directly in Excel.
StringIO:
Similar to BytesIO for PDFs, StringIO lets us build the CSV as a string in memory before sending it as a download, without ever saving it to disk.

8. Simulation and Demo Design

Key Viva Questions from Week 3
Q: What is Flask and why did you use it?
A: Flask is a lightweight Python web framework. I used it because it integrates directly with Python, making it easy to connect the AI models and database to a web interface without complex configuration.
Q: How does authentication work in your system?
A: When a user logs in, their credentials are verified against the database using SHA-256 hashed passwords. On success, Flask stores their username and role in an encrypted session cookie. Every subsequent request checks this cookie via the login_required decorator.
Q: How does the dashboard get live data?
A: The JavaScript in the dashboard calls the Flask API endpoints every few seconds using AJAX. The APIs read from the in-memory cache maintained by the CollectionEngine, which is updated every 3 seconds by a background thread.
Q: What happens when you simulate a failure?
A: The /api/simulate/failure endpoint saves extreme metric values to the database and updates the in-memory cache. The alert engine detects threshold violations on the next check cycle and creates CRITICAL alerts. The prediction engine scores the anomalous reading and updates the risk level. All of this propagates to the dashboard within seconds.
Q: Why generate a PDF report?
A: PDF reports provide a permanent, shareable record of infrastructure health that can be sent to management or archived. The CSV export enables further analysis in Excel or other tools.

📚 What You Should Learn From Week 4
1. HTML Structure
What HTML is:
HTML is the skeleton of every webpage. Every element on the page is defined by tags like <div>, <table>, <canvas>, <button>. The browser reads these tags and renders them visually.
Key HTML concepts used this week:

<canvas> — a blank drawing surface that JavaScript draws on. Used for the Digital Twin Map. The entire network topology is drawn using Canvas 2D API commands like ctx.arc(), ctx.lineTo(), ctx.fillText()
<table> — used for alerts table, comparison table, health table. Rows are <tr>, headers are <th>, cells are <td>
<script> — JavaScript code lives inside this tag at the bottom of each HTML file
Jinja2 templating — Flask passes variables like {{ user }} from Python to HTML. We used this to pass the logged-in user's data to each page


2. CSS — Styling and Layout
CSS Variables:
We defined all colours once at the top of every file inside :root {} using --variable-name syntax. This means changing one line changes the colour everywhere. This is why all 7 pages have exactly the same dark theme.
Key CSS concepts used:

display: flex — used for sidebar layout, nav items, metric rows. Flex makes elements sit side by side and align properly
display: grid — used for server cards grid (3 columns), stats grid (4 columns), charts grid (2 columns). Grid divides space into columns and rows automatically
position: fixed — used for the sidebar so it stays in place when you scroll
@keyframes — defines animations. The blinking dot, pulsing logo, floating orbs and card entrance animation all use keyframes
transition — smooth changes when you hover over buttons and cards
CSS pseudo-elements ::before — used for the coloured top border strip on cards without adding extra HTML
z-index — controls which element appears on top of another. Sidebar has z-index 100 so it's always above page content


3. JavaScript — Making Pages Dynamic
The fetch API:
Every page uses fetch() to call your Flask APIs and get live data. This is called AJAX — the page doesn't reload, only the data updates. For example:
javascriptconst res  = await fetch('/api/overview');
const data = await res.json();
This calls your Flask /api/overview endpoint and converts the JSON response into a JavaScript object.
async and await:
These keywords handle asynchronous operations. When you call fetch() it takes time to get a response. await pauses execution until the response arrives without freezing the browser. async marks a function that contains await.
setInterval:
Used on every page to call fetch functions repeatedly:
javascriptsetInterval(fetchOverview, 3000);
This calls fetchOverview every 3000 milliseconds (3 seconds), creating the live updating effect.
DOM manipulation:
document.getElementById('healthScore').textContent = health + '%' — this finds an HTML element by its id and changes its text. This is how the charts and numbers update without reloading the page.

4. Chart.js Library
What Chart.js is:
Chart.js is a JavaScript library that draws beautiful charts on <canvas> elements. We loaded it from a CDN (Content Delivery Network) with one script tag.
Chart types used:

type: 'line' — CPU history, RAM history charts on servers page. Shows data over time as a connected line
type: 'bar' — CPU comparison charts. Shows values as vertical bars
type: 'radar' — AI model comparison on AI Models page. Shows multiple values as a polygon shape

Key Chart.js concepts:

labels — the X axis values (timestamps, server names)
datasets — the actual data arrays
chart.update('none') — updates the chart without animation (important for live data so it doesn't flicker)
maintainAspectRatio: false — lets you control chart height with CSS


5. Canvas 2D API (Digital Twin Map)
What the Canvas API is:
The Digital Twin Map doesn't use Chart.js — it's drawn entirely with raw Canvas 2D API commands. This gives complete control over every pixel.
Key Canvas concepts:

ctx.beginPath() / ctx.arc() — draws circles for server nodes
ctx.lineTo() — draws lines for connections between servers
ctx.fillText() — draws text labels and icons
ctx.createRadialGradient() — creates the glowing effect around nodes
requestAnimationFrame(animate) — calls the draw function 60 times per second creating smooth animation
canvas.getBoundingClientRect() — converts mouse click coordinates to canvas coordinates for node selection


6. UI/UX Design Concepts
Colour coding for risk levels:
Every page uses the same colour system consistently: Green = OK/Low, Yellow = Warning/Medium, Orange = High, Red = Critical. This is called a visual hierarchy — users understand the severity instantly without reading.
Dark theme design:
The dark theme uses CSS variables with carefully chosen colours. The background (#020818) is near-black, cards (#0d1f3c) are slightly lighter, and the accent (#00d4ff) cyan provides contrast. This creates depth through layering.
Micro-interactions:
Small animations that give feedback — the button hover lift (transform: translateY(-2px)), the shake animation on wrong password, the toast notification sliding up, the server card border glow on hover. These make the interface feel alive and professional.
Responsive layout:
Using CSS Grid with repeat(3, 1fr) for server cards means 3 equal columns that automatically fill available space. This is better than fixed pixel widths.

7. Session Management in the Browser
How login persists across pages:
When you login, Flask sets a session cookie. Every page's JavaScript calls /api/me to get the current user's name and role. If the response says logged_in: false, the page redirects to /login automatically. This is why all 7 pages show your username in the sidebar.
Why logout clears everything:
The logout button calls POST /api/logout which tells Flask to clear the session cookie. After that, every API call returns 401 Unauthorized and the page redirects to login.

Key Viva Questions from Week 4
Q: What technology did you use to build the dashboard?
A: The dashboard is built with plain HTML, CSS and JavaScript — no frontend framework needed. Chart.js handles the charts, the Canvas 2D API handles the Digital Twin Map, and the fetch API handles live data updates from the Flask backend.
Q: How does the dashboard update in real time?
A: Each page uses JavaScript's setInterval to call the Flask API endpoints every 3 seconds. The responses update the DOM directly — changing text content, bar widths, chart data and colours — without reloading the page. This is AJAX-based live updating.
Q: What is the Digital Twin Map?
A: The Digital Twin Map is a visual representation of the entire IT infrastructure topology. Each server is a node positioned on a canvas, connected by lines that show data flow. Nodes are colour coded by risk level and pulse when critical. It's drawn using the HTML5 Canvas 2D API with requestAnimationFrame for smooth 60fps animation.
Q: How does the alert system work on the dashboard?
A: The alerts page calls /api/alerts every 3 seconds. When you click ACK or RESOLVE, it sends a POST request to the corresponding Flask endpoint which updates the database. The table re-renders automatically on the next fetch cycle showing the updated status.
Q: Why did you use Chart.js instead of building charts from scratch?
A: Chart.js is an industry-standard open-source library that handles cross-browser compatibility, responsive sizing, animations and accessibility automatically. Building equivalent charts from scratch would take weeks and produce inferior results. Using proven libraries is standard professional practice.
